SECTION 1: Introduction

Software to be Produced:

  • The software to be produced is a two-dimensional tank game application. The software provides a tank to be controlled by the user. The game will feature a multiplayer function. The software includes the functionality of an independently rotating, controllable turret head that fires at enemy tanks. Also included are premade maps.

Reference Documents:
  • Concept of Operations
  • Project Management Plan

Applicable Standards:
  • iOS 6.1
  • Android OS 4.0
  • Current generation web browsers

 Definitions, Acronyms, and Abbreviations:

  • None

SECTION 2: Product Overview

Assumptions:
  • The application will run on an android phone with the current OS or newer.
  • The application will run on an Apple iPhone with the current version of iOS or newer.
  • The application will be virtually be bug free and it will be stable.
  • We assume the user will have the correct devices to play the game; which will be either a smart phone for the applications or a computer with a keyboard for the web version.
  • We assume the user will have sufficient system requirements on which to run the application.

Stakeholders:
  • Professor and TAs
    •
    • These stakeholders will be analyzing the group information and development process throughout the course. They will also be grading us based upon said documentation and through presentations of the Tank Wars game.
  • Software Developers
    • These stakeholders will be responsible for the maintenance and handling of any problems, bugs, or fixes that will be needed before, during, or after the release of the Tank Wars game on the internet, current Android OS, and iOS.
  • Users
    • These stakeholders will be responsible for playing the game through their investments of time, be it short or long, and immersing themselves in the enjoyment of Tank Wars.
